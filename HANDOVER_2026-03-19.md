# WinP RDP 切断調査 引き継ぎメモ（2026-03-19）

## 目的
- 最小構成化ではなく、`ERRCONNECT_CONNECT_TRANSPORT_FAILED [0x0002000D]`（`BIO_read system error 54: Connection reset by peer`）の解消。

## 現在の症状
- 接続と認証は成功し、RemoteApp (`cmd.exe`) のウィンドウ生成まで進む。
- その後 1〜数秒でサーバー側から切断（RST）される。
- 代表ログ:
  - `ERRCONNECT_CONNECT_TRANSPORT_FAILED [0x0002000D]`
  - `disconnect_reason=rn-domain-disconnected`

## ここまでの主な実装変更

### 1) ログ強化
- `build=__DATE__ __TIME__` を出力。
- チャネル解決・RAIL処理の詳細ログを追加。

### 2) addin 解決の静的化対応（dlopen失敗対策）
- `winp_load_static_addin_entry` で以下を静的解決できるように実装:
  - static: `rdpdr`, `rail`, `cliprdr`, `drdynvc`, `rdpsnd`
  - dynamic/DVC: `rdpsnd`, `ainput`, `disp`
  - rdpsnd subsystem: `mac`, `fake`
- これで `/usr/local/lib/freerdp3/*.dylib` 不在による初期失敗は回避。

### 3) RAIL 起動シーケンス調整
- `ServerHandshakeEx` で `ClientInformation -> ClientSystemParam -> ClientExecute` を送信し `start cmd rc=0` を確認。
- `ClientSystemParam` の `workArea` を修正:
  - `right = width - 1`
  - `bottom = height - 1`

### 4) RemoteApp時の DisplayControl / 動的解像度抑止
- RemoteApp時に
  - `FreeRDP_SupportDisplayControl = FALSE`
  - `FreeRDP_DynamicResolutionUpdate = FALSE`
- `winp_loop()` で RemoteApp時は `pendingResize` が来ても monitor layout を送らない（完全スキップ）。
- ログ:
  - `[WinP] remoteapp display-control disabled`
  - `[WinP] remoteapp mode: suppress all resolution update requests`（初回のみ）

### 5) チャネル単位切り分けスイッチ追加（最新）
- 環境変数で個別に無効化可能:
  - `WINP_DISABLE_CLIPRDR=1`
  - `WINP_DISABLE_DRDYNVC=1`
  - `WINP_DISABLE_RDPSND=1`
- `winp_enable_audio_addins` と `winp_load_static_addin_entry` 両方で反映。
- 期待ログ例:
  - `audio-addins enabled: playback=... rdpsnd=... cliprdr=... drdynvc=...`
  - `addin-skip ... disabled by env`

## 最新ビルド
- `build=Mar 19 2026 00:54:xx` のバイナリが最新（このメモ作成時点）。

## 直近の観測ポイント
- RemoteApp・DisplayControl抑止後も `0x0002000D` は継続。
- 以前に `ERRCONNECT_CONNECT_CANCELLED [0x0002000B]` が混じった回は、再接続処理が重なったログで出た可能性あり（主問題は引き続き 0x0002000D）。

## 次回再開時の手順（優先）
1. `WINP_DISABLE_CLIPRDR=1` で実行して切断有無確認。
2. `WINP_DISABLE_RDPSND=1` で実行して切断有無確認。
3. `WINP_DISABLE_DRDYNVC=1` で実行して切断有無確認。
4. どの設定で安定化するかを基準に、原因チャネルを特定。
